# V4 自检结论

本代码已按以下原则修正：

1. 只使用 `.ioc` 中 TIM8_CH1/CH1N、TIM8_CH2/CH2N 对应的 PC6、PA7、PC7、PB0 作为全桥发波通道。
2. 主动关闭 `.ioc` 中已配置但本全桥未使用的 TIM8_CH3/CH3N、TIM8_CH4/CH4N。
3. 发波公式默认采用 testcore 的固定中心值 6784：
   `CCR1 = 6784 + 6784*m`，`CCR2 = 6784 - 6784*m`。
4. 若当前 TIM8->ARR 与 6784 不匹配，运行时安全回退到 `ARR/2`，并通过 `pwm_center_fallback` 报警。
5. 不包含 CubeMX 可生成的 SystemClock/GPIO/ADC/TIM/DMA 初始化代码。
6. 不再声称当前 `.ioc` 已经配置好 5 路 ADC；代码要求你补齐 ADC1 5-rank + DMA circular，或手动调用 `PVINV_LL_SetRawSamples()`。
7. 控制节拍默认外部调用，避免同时给出多个中断调用导致重复执行。
8. TIM8 Update Hook 和 ADC DMA Hook 只有在宏选择对应模式时才编译。
9. Break 故障不自动恢复，必须人工确认后调用 `PVINV_LL_ClearPwmBreakFaultAfterCheck()`。
10. 已用本地 stub 对 C 语法做过三种 tick-source 编译检查：EXTERNAL_ONLY、TIM8_UPDATE_HOOK、ADC_DMA_HOOK。

仍然必须由你实测确认：ADC 顺序、ADC 标定、电流方向、TIM8 PWM 频率、控制 ISR 频率、死区、Break 保护。电力电子项目无法仅凭代码保证硬件安全。
